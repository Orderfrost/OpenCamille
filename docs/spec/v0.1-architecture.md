# OpenCamille v0.1 Architecture Spec

> Updated: 2026-06-28

## Purpose

This spec is the build target for v0.1. If it conflicts with older notes, this
spec and `docs/architecture.md` are authoritative.

## Implementation Shape

Keep conceptual layers clear, but do not create empty classes or directories.

Recommended source layout:

```text
src/surfaces/cli.ts
src/runtime-control/session.ts
src/runtime-control/commands.ts
src/agent-runtime/loop.ts
src/agent-runtime/context.ts
src/services/tools.ts
src/services/skills.ts
src/services/memory.ts
src/services/mcp.ts
src/services/lifecycle.ts
src/services/hooks.ts
src/services/recorder.ts
src/infrastructure/provider.ts
src/infrastructure/config.ts
src/infrastructure/permissions.ts
src/infrastructure/mcp.ts
src/infrastructure/command.ts
src/infrastructure/workspace.ts
```

This is a guide, not a mandate. Merge files if implementation is tiny.

## Session

v0.1 keeps live state in one `Session` object.

```ts
type SessionStatus =
  | "idle"
  | "running"
  | "waiting_approval"
  | "paused"
  | "finished"
  | "failed"
  | "cancelled";
```

Session contains:

```text
sessionId
runId
status
conversationHistory
pendingApproval
budget
sessionSummary
```

Runtime Control is the only writer of `conversationHistory`.

## Provider Adapter

`ProviderAdapter` accepts provider-neutral `ContextAssembly` and produces
provider-neutral stream items.

```text
ProviderStreamItem:
  model_delta
  tool_use
  final_message
  usage
  error
```

Provider-specific SDK/API shapes must not leak into Agent Runtime.

Implement one real provider first and a fake provider for tests.

## ContextManager

Inputs:

```text
base prompt
project/user rules
active skill body
available skill metadata
tool schemas
session summary
persistent memory
recent ConversationHistory
```

Output:

```text
ContextAssembly
  system
  tools
  messages
```

If context is too large, return `context_overflow`. Do not silently truncate
inside ContextManager.

## AgentLoop

v0.1 loop:

```text
while not done:
  build context
  if context_overflow:
    Memory.compact()
    rebuild once

  ProviderAdapter.stream(context)
  emit model events
  collect final assistant message and tool_use items

  if tool_use:
    emit tool_requested
    result = Tools.runToolCall(toolUse)
    if approval_required:
      return pending approval to Runtime Control
    append normalized tool result through Runtime Control
    continue

  return final assistant message
```

Only AgentLoop emits turn/model/tool lifecycle events.

Do not implement separate `ToolExecutor`, `ExecutionStrategy`,
`TerminationGuard`, or `AgentRuntimeEvent`.

## Tools

`Tools` owns:

```text
register
list schemas
resolve
runToolCall
```

`runToolCall()`:

```text
validate args with zod
ask PermissionEngine
return approval_required if ask
execute callable if allow
normalize/truncate/redact result
return ok or error
```

`Tools` must not:

```text
mutate ConversationHistory
wait for user approval
write Recorder files directly
call LLM providers
decide next AgentLoop step
```

## Permission Flow

```text
Tools.runToolCall()
  -> PermissionEngine.authorize()
  -> allow | ask | deny
```

If `ask`:

```text
Tools.runToolCall()
  -> approval_required
AgentLoop
  -> returns pending approval
Runtime Control
  -> status = waiting_approval
Surface
  -> asks user
Runtime Control
  -> resumes same tool call after approval
```

`PermissionEngine` does not wait for users and does not store approval state.

## Lifecycle And EventBus

Lifecycle owns the event names and in-process EventBus.

EventBus interface can be minimal:

```ts
type EventBus = {
  emit(event: RunEvent): void | Promise<void>;
  subscribe(listener: (event: RunEvent) => void | Promise<void>): () => void;
};
```

Only AgentLoop and Runtime Control emit events.

Events include:

```text
session_created
session_resumed
session_closed
run_started
turn_started
context_built
context_overflow
model_started
model_delta
model_finished
model_failed
tool_requested
approval_requested
approval_resolved
tool_started
tool_finished
tool_failed
tool_denied
compaction_started
compaction_finished
compaction_failed
checkpoint_started
checkpoint_finished
checkpoint_failed
turn_finished
turn_failed
run_finished
run_failed
run_cancelled
```

Do not eventize internal debug details such as registry lookup attempts or token
estimator internals.

## Hooks

Hook points are placed in v0.1, but behavior is intentionally small.

Hook points:

```text
session_start
session_resume
session_stop
before_user_message
after_user_message
before_context_build
after_context_build
before_model_call
after_model_call
before_tool_call
after_tool_call
before_permission_request
after_permission_resolved
before_compaction
after_compaction
before_checkpoint
after_checkpoint
before_message_display
```

v0.1 behavior:

```text
observe-only by default
before_tool_call: deny or modify args
after_tool_call: modify result
before_user_message: deny or modify text
before_message_display: modify display only
```

Hooks are not implemented through EventBus when they need to block/modify flow.

## Recorder

Recorder writes:

```text
.opencamille/runs/<runId>/
  events.jsonl
  transcript.jsonl
  checkpoint.json
```

Rules:

```text
events.jsonl      primary timeline for eval/replay/debug
transcript.jsonl  readable conversation index
checkpoint.json   resume source
```

Recorder subscribes to EventBus for events. Runtime Control writes transcript
items and checkpoints at stable boundaries.

Do not use event sourcing for resume in v0.1.

## Skills

Follow Anthropic Agent Skills.

Minimum supported structure:

```text
skill-name/
  SKILL.md
```

`SKILL.md` must include frontmatter:

```yaml
name: skill-name
description: short trigger-oriented description
```

v0.1:

```text
Skills scans user/project directories
Skills builds SkillIndex
Tools registers built-in Skill tool
Skill tool activates skill body
ContextManager includes active skill body
```

Do not invent a custom skill package protocol.

## MCP

v0.1:

```text
Infrastructure/MCPClient:
  stdio JSON-RPC
  listTools
  callTool

Services/MCP:
  adapt MCP tools into Tools
```

No MCP HTTP transport, auth, resources, prompts, sampling, roots, elicitation,
or hot reload in v0.1.

## Memory

v0.1:

```text
SessionSummary
PersistentMemory
```

Compaction path:

```text
ContextManager -> context_overflow
AgentLoop -> Memory.compact()
ContextManager -> rebuild once
```

No `WorkingNotes`, retrieved memory, RAG, vector DB, or automatic long-term fact
extraction.

## Config

Config precedence, low to high:

```text
built-in defaults
env
user config
project config
CLI flags
```

Secrets:

```text
read from env/config
never write to events.jsonl
never write to transcript.jsonl
never write to checkpoint.json
redact known secret values from tool/model records
```

## Infrastructure Helpers

Use small files/functions:

```text
infrastructure/command.ts
  runCommand()

infrastructure/workspace.ts
  resolveWorkspacePath()
```

Do not create `FileSystem`, `Shell`, `Storage`, or fake `Sandbox` abstractions in
v0.1.

## Implementation Order

1. Config and core runtime types.
2. Lifecycle/EventBus.
3. Recorder.
4. Provider fake adapter.
5. Session and Runtime Control.
6. ContextManager.
7. Tools with validation, permissions, normalization, and redaction.
8. AgentLoop.
9. CLI surface.
10. One real ProviderAdapter.
11. Skills with built-in Skill tool.
12. stdio MCP tools.
13. Memory compaction.
14. Approval resume and checkpoint resume tests.
