02

横切模块

跨越多个层级，无层级访问限制，协调跨层关注面。

Lifecycle

统一生命周期系统：before 回调可阻断（deny/ask/allow），after 回调只观察（fire-and-forget）

spans: All layers · 节点: session/agent/turn/tool/subagent/memory

ContextManager

组装 ContextAssembly {system, reminders, tools, messages}，适配 Anthropic/OpenAI 注入差异

spans: Agent · MemoryStore · RuleLoader · SkillLoader

SkillLoader

Anthropic 标准渐进式加载：L1 元数据始终在上下文 → L2 触发时注入 → L3 按需读取

spans: Agent · Service · Infrastructure

ProviderAdapter

Streaming 归一化：Anthropic/OpenAI stream → StreamEvent。自定义通用 Message 格式

spans: Infrastructure → Agent (ThinkStep)

04

跨层依赖

正向依赖为主，显式声明的跨层直接调用（Agent → Infra 直达，跳过中间层）。

FromTo用途
Interface↓Session标准依赖链
Session↓Agent标准依赖链
Agent↓Service工具调用
Service↓Infrastructure工具实现依赖基础能力
Agent→Infra.ProviderAdapter模型调用直达（跨2层）
Agent→Infra.MemoryStore上下文组装（跨2层）
Session→Infra.SessionStore会话持久化（跨2层）
Infra.MemoryStore←Session.ConversationHistory反向只读数据流
05

MemoryStore · 3-Tier

MemoryStore 不持有原始 Message[] 副本，只存储衍生品。ConversationHistory 是唯一数据源。

Tier 1

Working Memory

Session.messages.slice(-N) 窗口引用。直接来自 Session，只读访问。

↳ Source: Session
Tier 2

Compressed Memory

CompactionAgent 生成 LLM 摘要。token > 80% 触发，Haiku 模型。失败 3 次后退化为规则截断。

↳ Generator: LLM (Haiku)
Tier 3

Persistent Memory

从 messages 提取关键事实 → ~/.camille/memory/*.md，跨会话持久化。

↳ Store: File System
