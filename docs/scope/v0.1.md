# OpenCamille v0.1 Scope

> 更新日期：2026-06-29  
> 作用：定义 v0.1 做什么、不做什么、如何验收。

## 目标

v0.1 的目标是构建一个最小但完整的 TypeScript Agent Harness 切片：

```text
CLI 输入
AgentLoop
Provider streaming
Tools.runToolCall()
Permission approval
ContextManager
Memory compaction
Skills
stdio MCP tools
Lifecycle/EventBus
Hooks 埋点
Recorder
Checkpoint resume
```

v0.1 不追求成为强 Coding Agent，也不追求完整产品形态。它要证明 OpenCamille 具备完整底层框架开发能力。

## In Scope

### Surface

```text
仅 CLI
streaming 输出
approval prompt
基本 command 输入
```

### Runtime Control

```text
单 live Session
ConversationHistory 唯一写入
CommandDispatcher
Session status / pendingApproval / budget
approval pause/resume
从 checkpoint 恢复 Session
```

### Agent Runtime

```text
ReAct-style AgentLoop
ContextManager
ProviderAdapter streaming contract
context_overflow -> Memory.compact once -> retry once
inline maxTurns / budget / abort checks
```

### Tools

```text
Tools.register
Tools.list schemas
Tools.resolve
Tools.runToolCall
zod runtime validation
permission gate
tool result normalization/truncation/redaction
```

内置工具：

```text
read file
search files
write/edit file
shell command
built-in Skill tool
MCP adapted tools
```

### Permissions

```text
PermissionEngine returns allow | ask | deny
Runtime Control owns pendingApproval
write/shell/MCP side-effect tools go through permission
approval_required can resume same tool call
```

### Skills

```text
遵循 Anthropic Agent Skills
读取 SKILL.md metadata
维护 SkillIndex
通过 built-in Skill tool 激活 skill body
支持 user-level 和 project-level skill directories
支持 manual reload
```

### MCP

```text
stdio MCP servers only
list tools
call tools
adapt MCP tools into Tools
走同一 permission / normalization / recorder 路径
```

### Memory

```text
SessionSummary
PersistentMemory as explicit local markdown
context overflow compaction
```

不做 RAG 或向量检索。

### Lifecycle / Hooks / Plugins

```text
in-process Lifecycle/EventBus
Run timeline events
HookPoint 全部埋点
Hooks 默认只观察
before_tool_call 可 deny/modify args
after_tool_call 可 modify result
before_user_message 可 deny/modify text
before_message_display 只允许修改展示内容
```

Plugins 顶级架构保留，但 v0.1 不做完整插件包格式。最多允许本地 config 注册 hooks/tools/skills。

```text

```

语义：


## 成功标准

v0.1 完成时必须满足：

1. 用户能启动 CLI session。
2. 用户能看到 provider streaming 输出。
3. Agent 能通过 ContextManager 构建厂商无关 context。
4. Agent 能通过 ProviderAdapter 调用一个真实 provider。
5. Agent 能调用 built-in tools。
6. Tool input 使用 zod 校验。
7. write/shell tool 能暂停 approval，并恢复同一个 tool call。
8. Tool output 在进入模型、Transcript、Events 前被规范化、截断、脱敏。
9. Skill 能被发现，并通过 built-in Skill tool 激活。
10. stdio MCP tool 能被 list/call，并走正常 tool path。
11. context overflow 能触发一次 compaction 和一次 retry。
12. Recorder 写出 `events.jsonl`、`transcript.jsonl`、`checkpoint.json`。
13. Resume 使用 `checkpoint.json`，不靠 event sourcing。
14. 核心逻辑有最小测试。

## Demo 目标

默认 demo 应该保持本地、小而完整：

```text
启动 CLI
打开一个小 repo
让 Agent inspect repo
stream model response
激活一个 Skill
读取/搜索/编辑文件
写入或 shell 前请求 approval
调用一个 stdio MCP tool
生成可检查的 events/transcript/checkpoint
```

Demo 证明 Harness，不和成熟 Coding Agent 比能力。
