# OpenCamille v0.1 Scope

> Updated: 2026-06-28

## Goal

Build the smallest complete TypeScript Agent Harness slice that can run a CLI
agent loop, stream output, call tools safely, manage context, record runs, and
resume from checkpoints.

v0.1 proves the harness. It does not compete with mature coding agents.

## In Scope

### Surface

- CLI only.
- Streaming output from lifecycle events.
- Approval prompt for permission requests.

### Runtime Control

- Single live session.
- ConversationHistory ownership.
- CommandDispatcher for user commands.
- Session status, pending approval, and budget fields in one `Session` object.
- Approval pause/resume for tool calls.

### Agent Runtime

- ReAct-style `AgentLoop`.
- `ContextManager` builds provider-neutral `ContextAssembly`.
- Provider streaming contract.
- Context overflow path: compact once, retry once.
- Inline guard checks for max turns, budget, and abort.

### Tools

- `Tools` owns register/list/resolve/runToolCall.
- Runtime validation with `zod`.
- Built-in tools:
  - read file
  - search files
  - write/edit file
  - shell command
  - built-in `Skill` loader tool
- Tool result normalization, truncation, and redaction.
- `Tools.runToolCall()` returns `ok`, `error`, or `approval_required`.

### Permissions

- `PermissionEngine` returns `allow | ask | deny`.
- It does not wait for users and does not own approval state.
- Runtime Control owns pending approval.
- Write and shell actions require permission handling.

### Skills

- Follow Anthropic Agent Skills.
- Load `SKILL.md` metadata into `SkillIndex`.
- Activate skill body through built-in `Skill` tool.
- Support `name`, `description`, and basic allowed tool behavior.
- User-level and project-level skill directories.
- Manual reload command.

### MCP

- stdio MCP servers only.
- list tools.
- call tools.
- adapt MCP tools into `Tools`.
- MCP tools use the same validation, permission, normalization, and recording
  path as built-in tools.

### Memory

- `SessionSummary`.
- `PersistentMemory` as explicit local markdown.
- No RAG or vector search.

### Lifecycle, Hooks, Plugins

- In-process Lifecycle/EventBus.
- Run timeline events for streaming, recording, replay, and eval evidence.
- Hook points are placed.
- v0.1 hook behavior is minimal:
  - observe-only by default
  - `before_tool_call` can deny/modify args
  - `after_tool_call` can modify result
  - `before_user_message` can deny/modify text
  - `before_message_display` can modify display only
- Plugins are top-level architecture only.
- v0.1 does not implement a plugin package format.
- v0.1 may allow local config to register hooks/tools/skills.

### Recorder

- File-based run records:

```text
.opencamille/runs/<runId>/
  events.jsonl
  transcript.jsonl
  checkpoint.json
```

- `events.jsonl`: primary timeline for eval/replay/debug.
- `transcript.jsonl`: readable conversation index.
- `checkpoint.json`: resume source.

### Infrastructure

- `ProviderAdapter`.
- `Config` with deterministic precedence and secret redaction requirements.
- `PermissionEngine`.
- `MCPClient`.
- `CommandRunner` as `runCommand()`.
- `WorkspacePath` as `resolveWorkspacePath()`.

## Out Of Scope

- Web UI.
- API surface.
- IDE integration.
- Subagents / multi-agent orchestration.
- RAG / vector memory / retrieved memory.
- Full plugin package format or marketplace.
- Skill fork subagents, dynamic command injection, skill-scoped hooks, live file
  watching, nested monorepo skill discovery.
- MCP resources, prompts, HTTP transport, OAuth/auth, sampling, roots,
  elicitation, hot reload.
- OpenTelemetry export.
- SQLite or remote storage.
- Real OS/container sandbox.
- Multi-provider parity.

## Success Criteria

v0.1 is complete when:

1. A user can start a CLI session and receive streaming output.
2. The agent can build context and call one real provider through
   `ProviderAdapter`.
3. The agent can call built-in tools through `Tools.runToolCall()`.
4. Write/shell tool calls can pause for approval and resume the same call.
5. Tool outputs are normalized, truncated, and redacted before recording or
   model re-entry.
6. A skill can be discovered and activated through the built-in `Skill` tool.
7. A stdio MCP tool can be listed and called through the normal tool path.
8. Context overflow triggers one compaction and one retry.
9. `events.jsonl`, `transcript.jsonl`, and `checkpoint.json` are written.
10. Resume uses `checkpoint.json`, not event sourcing.
11. Core behavior has tests for config precedence, permissions, tool validation,
    provider stream normalization, recorder serialization, and approval resume.

## Demo Target

The default demo should be local:

1. Start CLI.
2. Ask the agent to inspect a small repo.
3. Stream the model response.
4. Activate a skill.
5. Read/search/edit files.
6. Ask approval before write/shell actions.
7. Call one MCP stdio tool.
8. Produce run records that can be inspected for eval/replay/debug.
